# invect-xss-report
Технический отчет о критической уязвимости Reflected XSS (CWE-79) на поисковом веб-ресурсе invect.com. Описание вектора атаки, Proof of Concept (PoC) и анализ рисков безопасности в контексте серых рекламных фидов.
# Отчет об уязвимости: Отраженный XSS на сайте invect.com

**Дата обнаружения:** 8 июня 2026 г.  
**Тип уязвимости:** CWE-79 (Cross-site Scripting / Межсайтовый скриптинг)  
**Классификация OWASP:** A03:2021 – Injection  
**Исследователь:** PsyStudio  
**Статус:** Независимое исследование (Full Disclosure)

---

## 1. Краткое описание
В ходе независимого анализа параметров поиска на домене `invect.com` была обнаружена уязвимость Reflected Cross-Site Scripting (XSS). Веб-приложение некорректно фильтрует входящие данные в HTTP GET-запросе, что позволяет выполнять сторонний JavaScript-код в браузере пользователя.

## 2. Объект исследования
* **Уязвимый URL:** `https://invect.com`
* **Параметр:** `q` (строка поискового запроса)
* **Рекламный идентификатор:** `s=7801218269581587`

## 3. Технический анализ и Proof of Concept (PoC)
Уязвимость существует из-за отсутствия кодирования вывода. Символы `< > " '` не преобразуются в безопасные HTML-сущности.

### Вектор атаки (Пример запроса):
```http
GET /_search_results?s=7801218269581587&q=%3Cscript%3Ealert(document.domain)%3C/script%3E HTTP/1.1
Host: invect.com
```

### Механизм работы:
1. Браузер отправляет запрос с кодом внутри параметра `q`.
2. Сервер возвращает этот код обратно в теле HTML-страницы без изменений.
3. Браузер исполняет тег `<script>`, запуская сторонний сценарий.

## 4. Возможные риски
* **Кража сессий:** Доступ к куки (Cookie) и локальному хранилищу браузера.
* **Фишинг:** Подмена содержимого страницы и создание ложных форм ввода данных.
* **Перенаправления:** Принудительный редирект пользователей на опасные сайты.

## 5. Рекомендации по устранению
1. Внедрить HTML-кодирование (превращать `<` в `&lt;`) для всех выводимых данных пользователя.
2. Настроить политику безопасности CSP для запрета инлайновых скриптов:
```http
Content-Security-Policy: default-src 'self'; script-src 'self';
```
